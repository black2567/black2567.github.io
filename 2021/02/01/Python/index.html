决定报告哪些错误
在什么情况下该向用户报告错误？在什么情况下又应该在失败时一声不吭呢？如果用户知道要分析哪些文件，他们可能希望在有文件没有分析时出现一条消息，将其中的原因告 诉他们。如果用户只想看到结果，而并不知道要分析哪些文件，可能就无需在有些文件不存在时告知他们。向用户显示他不想看到的信息可能会降低程序的可用性。Python的错误 处理结构让你能够细致地控制与用户分享错误信息的程度，要分享多少信息由你决定。

通过模块json 能够将简单的Python数据结构转储到文件中，当程序再次运行时加载该文件中的数据。使用json 还可以在Python程序之间分享数据。更重要的是，JSON数据格式并非Python专用的，以JSON格式存储的数据能够与使用其他编程语言的人分享。
使用json.dump()和json.load()
json.dump()来存储
json.load()来读取

安装Pygame
先安装pip
访问https://bootstrap.pypa.io/get-pip.py，将页面代码复制，
并将其保存为get-pip.py文件，
命令：python get-pip.py 进行安装
在Windows中检查是否安装了pip
python -m pip –version
再安装Pygame
访问https://bitbucket.org/pygame/pygame/downloads/或http://www.lfd.uci.edu/~gohlke/pythonlibs/#pygame下载文件
命令：python -m pip install –user pygame-1.9.4-cp36-cp36m-win_amd64.whl
当报错：is not a supported wheel on this platform
首先确定python与pygame的版本是否对应，py3.6只能对应cp36
其次.whl命名问题也会导致
通过以下代码查看支持的命名名称
版本为AMD64
import pip._internal
print(pip._internal.pep425tags.get_supported())
版本为WIN32
import pip
print(pip.pep425tags.get_supported())
